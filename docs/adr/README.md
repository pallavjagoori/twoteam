# Architecture decision records

Architecture decision records explain choices that materially affect Twoteam's
compatibility, security, data model, operations or upstream-update process.

## Format

Each record contains:

- status
- context
- decision
- consequences
- alternatives considered

Use sequential names such as `0002-realtime-transport.md`. Accepted decisions
are not rewritten to hide history. Replace them with a newer decision when the
architecture changes.
